glxDetectOutput

The glxDetectOutput function detects if any or the specified hardware/driver combination is present. If so the glxOutput structure will be filled according to the actual hardware/driver capabilities.

int glxDetectOutput
(
int CardType,
int Base
)

Parameters

CardType

Device to detect. The following values are defined :

GLX_AUTODETECT

Use whatever device/driver is available

GLX_DIRECTSOUND

Use Directsound(tm) ONLY for sample output

GLX_WAVEDRIVER

Use Windows(r) Wave drivers ONLY for sample output

GLX_NOSOUND

Use nothing for sample output (all further sound-related calls can be made as normal)

Base

Reserved, must be zero

Return values

Returns GLXERR_NOERROR and a filled SoundCard structure if successful or one of the following errors otherwise :

GLXERR_OUTPUTACTIVE

GLXERR_UNSUPPORTEDDEVICE

glxStartOutput

The glxStartOutput function initialises the DSP-Core and audio hardware/drivers first. Then fills the glxOutput and glxBuffer structures with the correct values and finally starts outputting the actual sample-stream. A call to glxStartOutput must be matched by a call to glxStopOutput.

int glxStartOutput
(
void *Owner,
int Rate,
int Format,
int Mixahead
)

Parameters

Owner

Owner window of sample-stream, needed for low latency DirectSound(tm) output. If a NULL pointer is supplied low latency Directsound(tm) output will NOT be used, even when available.

Rate

Framerate to use for sample-stream in Hertz (8000..48000 Hz)

Format

Format to use for sample-stream. The following values are defined :

GLX_MONO

Use one (1) channel for output (monophonic output)

GLX_STEREO

Use two (2) channels for output (stereophonic output)

GLX_8BIT

Use eight (8) bits for output samples (48 dB S/N Ratio)

GLX_16BIT

Use sixteen (16) bits for output samples (96 dB S/N Ratio)

GLX_SHOLD

Use sample and hold for sample pitch shifting (no filtering)

GLX_COSINE

Use adaptive second order filtering for sample pitch shifting

GLX_2DAUDIO

Use two-dimensional audio routines (stereo panning)

GLX_3DAUDIO

Use three-dimensional positional audio routines (Galaxy software surround or hardware 3D positional audio)

Mixahead

Mixahead in milliseconds for the soundeffects. (5..495 msec)

Return values

Returns GLXERR_NOERROR if successful or one of the following errors otherwise :

GLXERR_OUTPUTACTIVE

GLXERR_UNSUPPORTEDFORMAT

GLXERR_DEVICEBUSY
